如何搭建Hexo+Github博客

Veröffentlicht am 2018-09-28 | Edited on 2018-09-29

前言

一个前端小白,用来记录前端学习历程,在网上查了诸多资料,搭建起了独立的博客,独立的才是自己的.
现在写下此文,分享给大家,过程很繁琐,耐住性子.

环境搭建

Node.js 的安装和准备

  • Node.js 去官方网站 根据操作系统去下载即可
  • 然后一路next 傻瓜式安装

  • 打开cmd黑窗口 验证是否安装成功

    • 验证node

    • 验证npm

  • 返回版本号即为安装成功,npm是和node一块安装上的

git 的安装和配置

  • git 去官方网站 根据操作系统去下载即可
  • 同上 傻瓜式安装
  • 在桌面空白处右击 出现 Git Bash Here 和 Git GUI Here 即可

github 账户的配置

  • github官网 注册账号
  • 登录后点击右上角加号 选择 New repository 创建代码库
  • 在Repository name下填写yourname.github.io, yourname必须和你的账户名保持一致 Description (optional)下填写一些简单的描述（不写也没有关系）

安装Hexo

安装

  • 在自己认为合适的地方创建一个文件夹，这里我以D：/blog 为例子讲解，首先在E盘目录下创建blog文件夹，并在命令行的窗口进入到该目录

  • 执行下面的命令,等待安装完成

  • 验证是否安装成功,执行下面的命令,返回一大串版本 如下图

配置

  • 初始化文件夹 等待完成

  • 启动本地服务器 出现下图即可

初体验

  • 在浏览器中地址栏中输入 http://localhost:4000/
  • 如图看到的及时hexo的初始化页面

托管到Github

配置Git个人信息

  • 设置Git的user name和email：(如果是第一次的话)

  • 生成ssh密匙

配置仓库信息

  • 找到 E:/blog/_config.yml 配置文件的最后一项 Deployment
  • 将配置修改为

    deploy:
    type: git
    repository: git@github.com:yourgithubname/yourgithubname.github.io.git
    branch: master

上传到github

  • 部署文件准备上线

小几率会报错 如果报错执行 npm i hexo 然后重新执行 hexo g

  • 正式上传

大几率会报错 如果报错执行 npm i hexo-deployer-git 然后重新执行 hexo g 和 hexo d

  • 上传成功 在github中会看到和博客相关的项目中有了好多文件
  • 线上测试 在浏览器输入yourgithubname.github.io 网址即可

    注意事项: 因为 github 的服务器也在国外 可能需要几分钟开会更新

写博客、发布文章

新建文章

这时候在我的电脑的目录下 E:\blog\source\ _posts 将会看到 article title.md 文件 编辑即可

发布文章

  • hexo g // 生成
  • hexo d // 部署
  • 上面两条命令可以简写成 hexo d -g

更换皮肤

  • 百度一下，找到一个合适的皮肤主题
  • 克隆下来以后是一个 themes 文件夹，我们要找到这个文件夹里面的内容- 找主题名称的文件夹（next） 以next主题为例
  • 把 next 这个文件夹放在我们博客的目录内blog/themes
  • 设置主题，来到 config.yml 文件内部（这个是对 hexo 进行的配置）
  • 找到 theme: landscape
  • 把后面的名称改成你的主题名称 next
  • 在 themes/next/config.yml 的文件,这个文件是对主题进行的一些配置,主题多种多样,配置方式多种多样,参考相关文档即可

Markdown简介及常用语法

Veröffentlicht am 2018-09-28 | Edited on 2018-09-29

简介

  • Markdown 是一种轻量级的「标记语言」，它的优点很多，目前也被越来越多的写作爱好者，撰稿者广泛使用。看到这里请不要被「标记」、「语言」所迷惑，Markdown 的语法十分简单。常用的标记符号也不超过十个，这种相对于更为复杂的 HTML 标记语言来说，Markdown 可谓是十分轻量的，学习成本也不需要太多，且一旦熟悉这种语法规则，会有一劳永逸的效果。

优点

  • 专注你的文字内容而不是排版样式，安心写作。
  • 轻松的导出 HTML、PDF 和本身的 .md 文件。
  • 纯文本内容，兼容所有的文本编辑器与字处理软件。
  • 随时修改你的文章版本，不必像字处理软件生成若干文件版本导致混乱。
  • 可读、直观、学习成本低。
